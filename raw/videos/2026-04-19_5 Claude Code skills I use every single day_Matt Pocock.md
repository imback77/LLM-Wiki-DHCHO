## 핵심 요약
AI 에이전트(Claude Code)의 결과물 품질을 극대화하기 위해 실제 개발 업무에 매일 사용하는 5가지 핵심 스킬(프롬프트 프로세스)을 소개합니다. AI에게 명확한 방향성과 제약을 부여하여 아이디어 구상부터 요구사항 정의(PRD), 이슈 분할, TDD 기반 개발, 코드베이스 아키텍처 개선까지의 전체 소프트웨어 개발 워크플로를 자동화하는 방법을 배울 수 있습니다.

---

## 주요 내용 (타임스탬프)
- [00:00:00] AI 에이전트 시대에 명확하고 엄격한 프로세스(Process)가 중요한 이유
- [00:01:23] **1. Grill me:** 코드를 작성하기 전, AI가 사용자를 끊임없이 인터뷰하여 요구사항의 의존성을 해결하고 완벽한 상호 이해에 도달하게 하는 스킬
- [00:04:06] **2. Write a PRD:** 'Grill me'로 합의된 아이디어를 바탕으로, 상세한 제품 요구사항 정의서(PRD)와 유저 스토리를 자동 작성하는 스킬
- [00:06:24] **3. PRD to issues:** 작성된 PRD를 AI가 독립적으로 실행 가능한 '수직적 슬라이스(Vertical slices)' 형태의 개별 GitHub 이슈로 잘게 분할하는 스킬
- [00:09:17] **4. TDD (Test-Driven Development):** AI가 Red-Green-Refactor 루프를 따르도록 강제하여, 테스트 코드를 먼저 작성하고 인터페이스를 설계하게 만들어 코드 품질을 높이는 스킬
- [00:12:33] **5. Improve codebase architecture:** 코드베이스를 분석하여 리팩토링 기회를 찾고, 서브 에이전트들을 생성해 여러 대안적 인터페이스 디자인을 제안받아 아키텍처를 개선하는 스킬

## 나의 생각·코멘트
AI는 기억력이 없고 제약이 없으면 엉뚱한 방향으로 가기 쉽기 때문에, 개발 워크플로를 여러 단계로 쪼개어 각 단계마다 명확한 역할(스킬)을 부여하는 접근법이 매우 인상적이다. 특히 코드를 바로 짜게 하지 않고 'Grill me' 스킬로 기획을 탄탄하게 다지는 부분과, 생성된 PRD를 작은 이슈 단위로 나누어 AI가 하나씩 해결하게 하는 방식은 내가 구축하려는 자동화 파이프라인이나 LLM 기반 프로젝트에도 바로 접목해 볼 만한 훌륭한 전략이다.
