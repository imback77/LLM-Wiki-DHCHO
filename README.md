# 🧠 LLM Wiki DHCHO (나만의 AI 세컨드 브레인)

> **"지식을 복리로 축적하고, 미래의 강력한 AI와 연결하는 개인 지식 저장소"**

이 저장소는 **후니(DHCHO)님**을 위한 맞춤형 AI 기반 개인 지식 관리 시스템(PKM)이자 세컨드 브레인(Second Brain)입니다. Andrej Karpathy가 제안한 **LLM Wiki 패턴**을 기반으로 설계되었으며, 단순히 기록을 남기는 것을 넘어 AI가 문맥을 파악하고 스스로 지식을 연결·통합(Synthesize)할 수 있는 구조로 세팅되었습니다.

---

## 🏗️ 전체 아키텍처 (3계층 구조)

> 지식의 원본 보존과 AI에 의한 가공 영역을 철저히 분리하는 것이 핵심입니다.

### 1. `raw/` 📥 (사용자 소유 - 원본 데이터 수집)
- **역할**: 원본 데이터를 **가공 없이** 저장하는 불변(Immutable) 영역.
- **하위 구조**:
  - `articles/` — 웹 클리핑, 언론 뉴스, AI 업계 동향
  - `videos/` — 유튜브, 영상 강의 학습 기록 (노트)
  - `research/` — 경제/투자 분석 리포트 및 시장 데이터 스냅샷
  - `notes/`, `courses/`, `conversations/` 등

### 2. `wiki/` 🤖 (AI 소유 - 지식 컴파일 및 그물망 형성)
- **역할**: AI가 정보를 **소화(Ingest)**하여 상호 연결된 지식 그래프를 구축하는 영역.
- **핵심 파일**:
  - `index.md`: 모든 지식 엔티티의 자동 갱신 목차. (**현재 170+ 페이지**)
  - `log.md`: 시스템 활동 및 인제스트 로그.
  - `2026-XX-XX_Market_Recap.md`: [New] AI가 생성한 일일 시장 리포트.

### 3. `output/` 📤 (최종 산출물)
- **역할**: 축적된 지식을 바탕으로 생성된 최종 결과물 (블로그 포스트, 보고서 등).

---

## 🚀 주요 모듈: 주식 시장 분석 엔진 (Market Analysis Engine)

현재 시스템은 단순한 지식 저장소를 넘어, **한국 주식 시장의 실시간 데이터를 분석하고 지식화하는 엔진**을 탑재하고 있습니다.

- **원클릭 업데이트 (`run_market_update.bat`)**:
    1. **시장 데이터 수집**: `FinanceDataReader` 및 `Naver Finance`를 통해 KOSPI/KOSDAQ 전 종목 데이터 확보.
    2. **트렌드 분석**: 거래대금 및 가격 모멘텀을 기반으로 실시간 주도 테마 추출 (`update_dynamic_trend.py`).
    3. **리스크 관리**: 매크로 지표(환율, 금리, VIX, SOX 등) 자동 추적 (`update_market_macro.py`).
    4. **데일리 리캡**: 수립된 데이터를 AI가 종합하여 `wiki/`에 리포트 자동 생성 (`ingest_daily_recap.py`).

---

## ⚙️ 시스템 지원 및 자동화

- **`clipper-templates/`**: 웹 수집을 위한 정해진 메타데이터 양식.
- **`skills/`**: AI 워크플로우를 자동화한 자산 (`/ingest`, `/fetch`, `/query` 등).
- **`tools/`**: 리포지토리 관리를 위한 헬퍼 스크립트 모음.
- **`graphify-out/`**: 지식 노드들의 매핑 상태를 HTML 및 데이터시트 형태로 렌더링.
- **`script.ps1`**: **토큰 다이어트(Token Diet)** 및 시스템 헬스 체크 메인 스크립트.

---

## 🚦 핵심 규칙 가이드라인

1. **지식 인제스트 (Ingest)**: 사용자가 `raw/`에 정보를 넣고 AI에게 요청하면, AI는 지식을 연결(Link)하고 엔티티 페이지를 갱신합니다.
2. **지식의 복리 성장**: 새로운 인풋은 반드시 `index.md`와 `log.md`를 거치며 기존 투자/경제/AI 관점과 융합(Synthesis)됩니다.
3. **토큰 절약 (Token Diet)**: AI에게 모든 데이터를 보내는 대신, 로컬 스크립트가 전처리한 요약 데이터만 전달하여 운영 비용을 최소화합니다.
