# /fetch 명령어: 웹 정보 스크랩 및 클리핑 봇

`/fetch <keyword>` 명령어나 "AI 기사 좀 가져와줘" 등의 자연어 요청 시 발동하는 스킬입니다.
안티그래비티의 웹 검색(`search_web`)과 페이지 내용 읽기(`read_url_content`) 능력을 통해, 사용자가 일일이 웹클리퍼를 누르지 않아도 **지정된 양식의 마크다운 파일**로 `raw/articles`에 자동 적재합니다.

---

## 1. 검색 및 타겟 설정

1. **타겟 기사 수**: 요청 당일 작성된 기사 중 **최신 10개**를 타겟으로 합니다.
2. **검색 소스 안내**: 현 단계에서는 일반 구글 검색 최상단 기사를 우선 가져옵니다. 
   > *💡 사용자가 검색 사이트 5개를 특정하는 경우 해당 도메인 위주로 쿼리를 보냅니다.*
3. 안티그래비티는 `search_web` 툴을 사용해 적절한 URL 10개를 획득합니다. (한 번에 못 찾을 시 2~3회 추가 검색 가능)

---

## 2. 콘텐츠 추출 및 템플릿 렌더링

`read_url_content` 툴을 이용해 각각의 기사 본문을 읽어 옵니다. 
추출된 내용은 **반드시** 아래의 `Obsidian Web Clipper` 아티클 템플릿 형식으로 작성되어야 합니다.

### [필수 템플릿 구조]

```markdown
---
type: "article"
title: "{기사의 실제 제목}"
source: "{기사 URL}"
author: ["{기사의 작성자나 매체 이름}"]
published: {기사 발행일, 예: 2026-04-16}
created: {스크랩한 오늘 날짜, 예: 2026-04-16}
description: "{기사의 짧은 1~2줄 핵심 요약}"
tags: ["raw/articles"]
wiki_status: "미처리"
---
## 핵심 요약
{description 내용과 동일하거나 약간 더 풀어쓴 본문 요약}

---

## 본문
{read_url_content로 가져온 기사 본문의 전체 텍스트를 마크다운으로 그대로 삽입}

---

## 나의 생각·코멘트
(이 글에서 얻은 나만의 인사이트를 적어주세요)

## 관련 키워드
(wiki 연결에 활용할 키워드)
```

> **주의사항**: 저작권 및 원본 보존 철학에 따라, **본문 내용은 가능한 자르지 않고 원문 그대로** 넣습니다. 유저가 이후 위키에 합칠지 말지 읽으면서 판단합니다.

---

## 3. 파일 저장 

- **위치**: `raw/articles/`
- **파일명**: `{오늘 날짜}_{기사 제목}.md` 
  - (예: `2026-04-16_OpenAI가 새로운 모델을 발표했습니다.md`)
  - 파일명에 사용할 수 없는 특수 문자(`:`, `?`, `/`, `\` 등)는 제외합니다.
- 안티그래비티는 `write_to_file` 명령어를 사용해 10개의 .md 파일을 일괄 생성합니다.

---

## 4. 후속 처리 안내 및 종료

모든 기사 저장이 완료되면, 터미널 명령을 통해 git에 커밋(`git add . && git commit -m "feat: fetch 10 articles on [키워드]"`)하여 로컬 백업을 진행합니다.
이후 사용자에게 다음과 같이 알립니다.
"10개의 기사를 raw/articles 에 저장했습니다. 읽어보시고 불필요한 파일은 삭제해주세요. 남은 파일들을 나중에 `위키에 반영해줘`라고 하시면 한 번에 Ingest 하겠습니다."
