# /query 스킬 — Wiki 기반 질문 답변

> 이 파일은 Antigravity가 읽고 실행하는 스킬 지침서입니다.
> 후니님이 `/query` 또는 질문을 하면, 벡터 DB가 아닌 **wiki/ 문서를 직접 읽고** 답변합니다.

---

## 트리거

다음 중 하나를 감지하면 이 스킬을 실행합니다:
- `/query 질문내용`
- "wiki에서 찾아줘"
- "내 위키 기반으로 답해줘"
- "기록된 거 참고해서 답해줘"

---

## 실행 절차

### STEP 1. 질문 파악

후니님의 질문에서 다음을 추출합니다:
- **핵심 키워드** — 질문의 핵심 단어 2~5개
- **질문 유형** 분류:
  - `개념 이해` — "~가 뭐야?", "~는 어떤 거야?"
  - `비교` — "~와 ~의 차이는?", "~중에 뭐가 나아?"
  - `방법` — "~을 어떻게 해?", "~하는 법"
  - `연결` — "~와 ~가 어떻게 연결돼?", "~에 대해 내가 뭘 알아?"
  - `종합` — "~에 대해 내가 아는 걸 정리해줘"

---

### STEP 2. index.md 탐색

`wiki/index.md`를 읽고:
1. 질문의 핵심 키워드와 관련된 페이지를 찾습니다
2. 관련 페이지 목록을 만들고 우선순위를 정합니다:
   - **직접 관련** (키워드가 페이지명에 포함)
   - **간접 관련** (요약에 키워드가 언급)
3. 관련 페이지가 없으면 → STEP 2-B 실행

**STEP 2-B. 관련 페이지 없을 때**
```
📭 wiki에서 관련 내용을 찾지 못했습니다.

현재 wiki 보유 지식:
- 개념: [목록]
- 엔티티: [목록]
- 요약: [목록]

다음 중 원하시는 것을 선택해주세요:
① 일반 지식으로 답변하기 (wiki 없이)
② 이 주제를 새로 수집해서 ingest하기
③ 직접 wiki 페이지를 만들어달라고 요청하기
```

---

### STEP 3. 관련 페이지 읽기

찾은 관련 페이지를 순서대로 읽습니다:
- 직접 관련 페이지 → **전체 내용 읽기**
- 간접 관련 페이지 → **핵심 섹션 읽기**
- 각 페이지의 `[[위키링크]]`를 따라가며 연결된 페이지도 확인

> 읽은 페이지 목록을 답변 하단에 `참고 wiki` 섹션으로 명시합니다.

---

### STEP 4. 답변 생성

읽은 wiki 내용을 바탕으로 답변을 작성합니다.

**답변 원칙:**
- wiki에 있는 내용 → **wiki 내용을 기반으로** 답변 (출처 명시)
- wiki에 없는 내용 → **일반 지식으로 보완**하되 명확히 구분:
  - `📗 wiki 기반:` — wiki에서 찾은 내용
  - `💭 일반 지식:` — wiki 외 일반 답변

**답변 형식 (질문 유형별):**

| 질문 유형 | 답변 형식 |
|-----------|-----------|
| 개념 이해 | 정의 → 핵심 포인트 → 후니님 맥락에서의 의미 |
| 비교 | 비교 표 → 후니님 상황에 맞는 추천 |
| 방법 | 단계별 순서 → 후니님 현재 세팅과 연결 |
| 연결 | 관계도 설명 → 새로운 인사이트 제안 |
| 종합 | 알고 있는 것 정리 → 빠진 부분 식별 |

**답변 말미에 항상 포함:**
```
---
📚 참고한 wiki 페이지:
- [[페이지명]] — (관련성 이유 한 줄)
- [[페이지명]] — (관련성 이유 한 줄)

📭 wiki에 아직 없는 관련 내용:
- (이 질문과 관련해 wiki에 없는 부분 언급)
```

---

### STEP 5. 인사이트 저장 여부 확인

답변이 가치 있는 종합·분석·연결인 경우, 답변 말미에 제안합니다:

```
💾 이 답변을 wiki 인사이트 페이지로 저장할까요?
→ "저장해줘"라고 하시면 wiki/인사이트_YYYY-MM-DD_주제.md 로 저장됩니다.
```

**인사이트 페이지 저장 시:**
- 경로: `wiki/인사이트_YYYY-MM-DD_주제.md`
- frontmatter: `type: insight`
- 내용: 질문 + 요약된 답변 + 참고 페이지
- index.md 인사이트 섹션에 추가
- log.md에 기록: `## [YYYY-MM-DD] query | 질문 요약`

---

## 규칙 및 제약

| 규칙 | 내용 |
|------|------|
| wiki 우선 | 항상 wiki/index.md를 먼저 읽은 후 답변한다 |
| 출처 명시 | wiki 기반 내용과 일반 지식을 명확히 구분한다 |
| raw/ 불변 | 이 스킬 실행 중 raw/ 파일은 수정하지 않는다 |
| 투명성 | 참고한 wiki 페이지를 항상 답변 하단에 나열한다 |
| 인사이트 제안 | 가치 있는 답변은 저장을 제안하되, 강요하지 않는다 |
| 언어 | 한국어 기본, 기술 용어는 영어 병기 |
