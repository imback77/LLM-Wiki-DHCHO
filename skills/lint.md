# /lint 스킬 — Wiki 건강 점검 & 정리

> 이 파일은 Antigravity가 읽고 실행하는 스킬 지침서입니다.
> 후니님이 `/lint` 를 호출하면, wiki/ 전체를 점검하고 문제를 수정·업데이트합니다.
> 도서관 사서처럼 — 책이 제자리에 있는지, 내용이 최신인지, 연결이 끊기지 않았는지 확인합니다.

---

## 트리거

다음 중 하나를 감지하면 이 스킬을 실행합니다:
- `/lint`
- "위키 점검해줘"
- "위키 정리해줘"
- "깨진 링크 찾아줘"
- "위키 건강 체크"

---

## 실행 절차

### STEP 1. 전체 파일 목록 수집

`wiki/` 폴더의 모든 `.md` 파일을 읽어 목록을 만듭니다.
(`ANTIGRAVITY.md` 제외)

현황을 먼저 보고합니다:
```
🔍 lint 시작 — wiki 현황
  총 페이지: N개
  개념(concept): N개 | 엔티티(entity): N개
  요약(summary): N개 | 비교(comparison): N개 | 인사이트(insight): N개
```

---

### STEP 2. 7가지 점검 항목 (순서대로 실행)

---

#### 🔴 점검 1 — 깨진 위키링크 (Broken Links)

각 페이지의 `[[위키링크]]`를 수집하고,
해당 링크 이름의 파일이 `wiki/` 폴더에 실제로 존재하는지 확인합니다.

**문제 유형:**
- 링크된 페이지가 아직 없음 → `🔴 미생성 링크`
- 오탈자로 인한 링크 불일치 → `🔴 이름 불일치`

**자동 처리:**
- 미생성 링크가 중요한 개념/엔티티라면 → 빈 스텁(stub) 페이지 자동 생성
  ```yaml
  ---
  type: stub
  created: YYYY-MM-DD
  note: "이 페이지는 다른 페이지에서 링크만 된 상태입니다. ingest 후 채워주세요."
  ---
  # 페이지명
  > 📌 이 페이지는 아직 내용이 없습니다. 관련 자료를 수집하면 자동으로 채워집니다.
  ```

---

#### 🟡 점검 2 — 고아 페이지 (Orphan Pages)

다른 어떤 페이지에서도 `[[링크]]`로 연결되지 않은 페이지를 찾습니다.

**처리:**
- 고아 페이지 목록을 후니님께 보고
- 관련성 있는 페이지를 찾아 `[[링크]]` 자동 추가 (가능한 경우)
- 자동 추가가 불가능한 경우 → 후니님께 수동 연결 제안

---

#### 🟡 점검 3 — index.md 누락 항목

`wiki/` 폴더의 실제 파일 목록과 `index.md`의 등록 목록을 비교합니다.

**문제 유형:**
- 파일은 존재하지만 index에 없는 페이지 → `🟡 index 미등록`
- index에는 있지만 파일이 없는 항목 → `🔴 파일 없음`

**자동 처리:**
- index 미등록 페이지 → index.md 해당 카테고리에 자동 추가
- 파일 없는 index 항목 → 삭제 후 후니님께 보고

---

#### 🟡 점검 4 — frontmatter 누락·불완전

각 페이지의 YAML frontmatter를 확인합니다.

**필수 필드:**
```yaml
type:     # concept | entity | summary | comparison | insight | stub
created:  # YYYY-MM-DD
updated:  # YYYY-MM-DD
tags:     # 최소 1개 이상
```

**자동 처리:**
- `updated` 날짜가 없거나 오래됐으면 현재 날짜로 업데이트
- `type`이 없으면 파일명·내용 기반으로 추론 후 추가
- `tags`가 없으면 내용 기반으로 태그 추론 후 추가

---

#### 🟠 점검 5 — 모순 감지 (Contradictions)

같은 개념·엔티티에 대해 여러 페이지에서 **서로 다른 내용**을 기술하는지 확인합니다.

**감지 방법:**
- 동일 주제를 다루는 페이지들을 함께 읽고 내용 비교
- 날짜·수치·설명이 충돌하는 부분 탐지

**처리:**
- 모순 발견 시 당사자 페이지에 표기:
  ```
  > [!WARNING] 모순 감지
  > 이 내용이 [[다른 페이지]]의 내용과 충돌합니다. 확인 필요.
  ```
- 후니님께 어떤 내용이 맞는지 확인 요청

---

#### 🔵 점검 6 — 누락된 개념 페이지 (Missing Concept Pages)

요약(summary) 페이지나 인사이트 페이지에서 **언급됐지만 아직 독립 페이지가 없는 개념**을 찾습니다.

**처리:**
- 중요도가 높은 누락 개념 → 스텁 페이지 자동 생성
- 목록을 후니님께 보고하며 ingest 추천

---

#### 🔵 점검 7 — 최신화 점검 (Staleness Check)

`log.md`를 읽어 **마지막 ingest 이후 오래된 페이지**를 찾습니다.

기준:
- 생성 후 30일 이상 업데이트 없는 페이지
- `sources`로 연결된 raw/ 파일이 수정됐지만 wiki는 미반영인 경우

**처리:**
- 오래된 페이지 목록 보고
- 업데이트 필요 여부를 후니님께 확인

---

### STEP 3. 자동 수정 실행

점검 결과를 바탕으로 **자동 처리 가능한 항목**을 먼저 수정합니다:

| 자동 처리 가능 | 후니님 확인 필요 |
|----------------|-----------------|
| index.md 미등록 항목 추가 | 모순 해결 |
| frontmatter 누락 필드 추가 | 고아 페이지 연결 방향 |
| updated 날짜 갱신 | 스텁 페이지 내용 채우기 |
| 스텁 페이지 생성 | 오래된 페이지 업데이트 여부 |
| index.md 통계 업데이트 | — |

## 규칙 및 제약

| 규칙 | 내용 |
|------|------|
| raw/ 불변 | lint 중 raw/ 파일은 절대 수정하지 않는다 |
| 자동 수정 우선 | 명확한 오류는 자동 수정하고 보고한다 |
| 모순은 확인 후 | 모순 해결은 반드시 후니님 확인 후 처리한다 |
| 삭제 금지 | 파일 삭제는 하지 않는다. 문제 있는 파일은 표기만 한다 |
| 스텁 생성 | 중요한 미생성 링크는 스텁 페이지로 자리를 잡아둔다 |
| 언어 | 한국어 기본, 기술 용어는 영어 병기 |
