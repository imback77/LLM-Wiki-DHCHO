# /ingest 스킬 — LLM Wiki Ingest 자동화

> 이 파일은 Antigravity가 읽고 실행하는 스킬 지침서입니다.
> 후니님이 `/ingest` 라고 말하면 이 스킬을 찾아 아래 절차를 실행하세요.

---

## 트리거

다음 중 하나를 감지하면 이 스킬을 실행합니다:
- `/ingest`
- "인제스트해줘"
- "raw에 새 파일 있어"
- "새로 클리핑한 거 정리해줘"

---

## 실행 절차

### STEP 1. raw/ 스캔 — 미처리 파일 탐지

아래 6개 폴더를 순서대로 확인합니다:
```
raw/articles/
raw/videos/
raw/research/
raw/courses/
raw/notes/
raw/conversations/
```

각 폴더에서 `ANTIGRAVITY.md`를 **제외한** 모든 `.md` 파일을 찾습니다.
찾은 파일의 frontmatter에서 `wiki_status: 미처리` 인 파일을 우선 처리합니다.
`wiki_status`가 없는 파일도 미처리로 간주합니다.

**결과 보고 형식:**
```
📂 미처리 파일 발견: N개
- [유형] 파일명 (폴더)
- [유형] 파일명 (폴더)
```

미처리 파일이 없으면:
```
✅ 모든 raw/ 파일이 처리 완료 상태입니다.
```

---

### STEP 2. 파일 읽기 (복수 파일 시 하나씩 순서대로)

> ⚠️ raw/ 파일은 절대 수정하지 않습니다.

파일을 읽고 다음을 추출합니다:

**A. 핵심 주장 (Key Claims)**
- 원본에서 가장 중요한 주장·정보 3~5개
- 원본 문장 그대로가 아닌, 핵심을 압축해서

**B. 언급된 엔티티 (Entities)**
- 사람 (인물, 저자, 연구자)
- 도구·제품·서비스
- 조직·기업·기관

**C. 다루는 개념 (Concepts)**
- 이 소스에서 설명하거나 활용하는 핵심 개념들
- 이미 wiki에 존재하는 개념이면 `[[링크]]` 형태로 표시

---

### STEP 3. wiki/ 즉시 반영 (질문 생략)

질문 없이 바로 STEP 4로 진행합니다.
소스의 타입, 맥락, 후속 액션은 소스 내용에서 추론하여 자동으로 wiki에 반영합니다.

---

### STEP 4. wiki/ 반영

후니님의 답변을 받은 후 아래를 실행합니다:

#### 4-1. 소스 요약 페이지 생성
- 경로: `wiki/YYYY-MM-DD_소스제목.md`
- frontmatter: type, created, updated, sources, tags
- 포함 내용: 핵심 주장, 엔티티, 개념, **후니님의 맥락 메모**, 후속 액션 체크리스트

#### 4-2. 엔티티 페이지 (entity)
- 경로: `wiki/엔티티명.md`
- 기존 페이지가 있으면 → **새 정보를 추가 업데이트**
- 없으면 → 새로 생성

#### 4-3. 개념 페이지 (concept)
- 경로: `wiki/개념명.md`
- 기존 페이지가 있으면 → **새 소스 정보로 보강**
- 없으면 → 새로 생성

#### 4-4. index.md 업데이트
- 새로 생성된 페이지를 해당 카테고리 테이블에 추가
- 통계 숫자 업데이트 (총 N개 페이지)
- `updated` 날짜 업데이트

#### 4-5. log.md 업데이트
- 다음 형식으로 append:
```
## [YYYY-MM-DD] ingest | 소스 제목
- **소스**: raw/폴더/파일명.md
- **타입**: article | video-note | research | course-note | conversation
- **생성된 페이지** (N개): 페이지 목록
- **후니님 맥락**: 수집 이유 / 연결점 / 다음 액션
```

---

### STEP 5. 완료 보고

```
✅ Ingest 완료

📄 처리한 소스: 파일명
📝 생성·업데이트된 페이지: N개
  - [신규] wiki/페이지명.md
  - [업데이트] wiki/페이지명.md
📊 wiki 현황: 총 N개 페이지

💡 다음 제안:
  - (이 소스와 연결 가능한 다른 wiki 페이지나 후속 액션 1~2개 제안)
```

---

## 규칙 및 제약

| 규칙 | 내용 |
|------|------|
| raw/ 불변 | raw/ 파일은 절대 수정·삭제하지 않는다 |
| 한 번에 하나 | 복수 파일이 있어도 하나씩 순서대로 처리한다 |
| 질문 필수 | STEP 3 질문 없이 wiki를 생성하지 않는다 |
| 링크 연결 | 모든 wiki 페이지는 `[[위키링크]]`로 상호 연결한다 |
| frontmatter | 모든 wiki 페이지에 YAML frontmatter를 포함한다 |
| 언어 | 한국어 기본, 기술 용어는 영어 병기 |
