## AI에게 기대하는 것

- 내가 배운 지식을 **잊지 않도록** 구조화하여 체계적으로 기록·정리를 도와줘
- 기술 용어는 **쉽게 풀어서** 설명해줘 — 코딩/개발 기초가 부족한 상태야
- 단순 답변이 아니라, **왜 그런지(Why)와 어떤 맥락인지(Context)**를 함께 알려줘
- 내 기록과 기존 지식을 **연결(Link)**해서 새로운 인사이트를 만들어줘
- 장기적 성장 관점에서 도움을 줘 — 즉각적인 답보다 **쌓이는 지식**이 중요해

## 위키 아키텍처 (LLM Wiki)

> Karpathy의 LLM Wiki 패턴을 후니님 맥락에 맞게 적용한 구조.
> 핵심 아이디어: RAG처럼 매번 재검색하지 않고, AI가 **지속적으로 축적·유지하는 위키**를 만든다.

### 3계층 구조

```
LLM Wiki DHCHO/
├── ANTIGRAVITY.md          ← 이 파일 (AI 지침서 = 스키마)
├── 나의 핵심 맥락.md         ← 나는 누구인가
│
├── raw/                    ← 🔒 불변 원본 (후니님이 관리)
│   ├── articles/           ← 웹 클리핑, 기사, 뉴스레터
│   ├── notes/              ← 개인 메모, 일지, 아이디어
│   ├── conversations/      ← AI 대화 기록
│   ├── courses/            ← 수업·강의 자료
│   ├── research/           ← 투자 리서치, 경제 보고서
│   ├── videos/             ← 유튜브·영상 학습 노트
│   └── assets/             ← 이미지, 스크린샷, 미디어
│
├── wiki/                   ← 🤖 AI가 컴파일하는 위키 (AI가 관리)
│   ├── ANTIGRAVITY.md      ← wiki 폴더 규칙
│   ├── index.md            ← 전체 목차 (카테고리별)
│   ├── log.md              ← 활동 기록 (시간순)
│   └── (위키 페이지들...)
│
└── output/                 ← 📤 최종 결과물
    ├── blog/               ← 워드프레스 블로그 글
    ├── reports/            ← 투자 분석·리서치 보고서
    ├── lectures/           ← 강의 교안·프레젠테이션
    └── misc/               ← 기타 결과물
```

### 역할 분담

| 계층 | 누가 소유 | 규칙 |
|------|-----------|------|
| **raw/** | 후니님 | 원본 자료를 넣는다. AI는 **읽기만** 하고 절대 수정하지 않는다 |
| **wiki/** | AI | raw를 읽고 요약·연결·종합한다. 후니님은 읽고 피드백만 준다 |
| **output/** | AI (요청 시) | wiki의 지식을 조합해 결과물을 만든다 |

## 위키 운영 규칙 (Operations)

### 📥 Ingest (수집)
후니님이 raw/에 새 자료를 넣고 "이거 정리해줘"라고 하면:
1. 원본을 읽고 핵심 내용을 후니님과 논의한다
2. wiki/에 **요약 페이지**를 생성한다
3. 관련 **기존 페이지를 업데이트**한다 (연결, 보강, 모순 표기)
4. `wiki/index.md`를 갱신한다
5. `wiki/log.md`에 기록한다: `## [YYYY-MM-DD] ingest | 제목`

### 🔍 Query (질문)
후니님이 질문하면:
1. `wiki/index.md`를 먼저 읽고 관련 페이지를 찾는다
2. 관련 위키 페이지를 읽고 종합하여 답변한다
3. 가치 있는 답변은 wiki에 **인사이트 페이지**로 저장할 수 있다
4. `wiki/log.md`에 기록한다: `## [YYYY-MM-DD] query | 질문 요약`

### 🔧 Lint (건강 점검)
후니님이 "위키 점검해줘"라고 하면:
1. 페이지 간 **모순** 확인
2. **고아 페이지** (다른 페이지에서 링크되지 않은 페이지) 탐지
3. 언급만 되고 아직 **페이지가 없는 개념** 식별
4. 누락된 **교차 참조** 제안
5. 새로 조사할 **질문이나 자료** 추천
6. `wiki/log.md`에 기록한다: `## [YYYY-MM-DD] lint | 점검 결과 요약`

## 위키 페이지 작성 규칙

### YAML Frontmatter (필수)
```yaml
---
type: concept | entity | summary | comparison | insight
created: YYYY-MM-DD
updated: YYYY-MM-DD
sources: [원본 파일 경로]
tags: [태그1, 태그2]
---
```

### 본문 규칙
- 제목은 `# 페이지 이름`
- `[[위키링크]]`로 관련 페이지를 적극 연결
- 모순 발견 시: `> [!WARNING] 모순 발견: ...`
- 출처 명시: `> 출처: [[raw/articles/2026-04-16_기사제목]]`
